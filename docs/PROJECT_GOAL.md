# Project Goal

## 1. 목표

이 프로젝트의 목표는 OpenRTB 2.6의 입찰 요청/응답 흐름을 기반으로, **실시간 광고 입찰 요청을 처리하고 제한 시간 안에서 낙찰자를 결정하는 저지연 RTB 입찰 시스템**을 단계적으로 구현하는 것이다.

이 프로젝트는 광고 도메인 전체를 구현하기 위한 것이 아니다. RTB의 응답 제한 시간과 불완전한 Bidder 응답이라는 제약을 이용해, 입찰 요청 처리부터 낙찰자 결정까지의 핵심 경로를 작게 정의하고 검증한다.

## 2. 핵심 문제

하나의 `BidRequest`가 들어왔을 때 여러 Bidder가 동시에 응답할 수 있다. 하지만 모든 Bidder가 빠르고 정상적으로 응답한다는 보장은 없다.

이 프로젝트에서 다루려는 핵심 문제는 다음과 같다.

> 대용량 입찰 요청 상황을 가정하고, 제한 시간 안에서 bid/no-bid 판단과 여러 BidResponse 수집을 수행한 뒤, timeout, invalid bid, late bid를 제외하고 유효한 bid 중 낙찰자를 결정하는 것.

## 3. 현재 범위

현재 목표 설정 단계에서는 아래 범위까지만 확정한다.

- OpenRTB 2.6의 `BidRequest` / `BidResponse` 흐름을 기반으로 한다.
- 시스템의 관심사는 입찰 요청 처리, bid/no-bid 판단, BidResponse 생성/수집, 낙찰자 결정까지이다.
- 실제 광고 노출, 클릭, 과금, 리포팅은 제외한다.
- 상세 컴포넌트 구조와 기술 선택은 Tech Spec 단계에서 결정한다.
- 최적화 방식은 초기 구현과 측정 이후 결정한다.

## 4. 기술적 초점

이 프로젝트는 다음 문제 축에 집중한다.

- OpenRTB 입찰 요청/응답 계약을 기반으로 한 요청 처리
- 제한 시간 안에서의 Bidder 응답 수집과 실패 케이스 분류
- 유효한 입찰만을 대상으로 한 낙찰자 결정
- 측정 가능한 성능 지표를 기반으로 한 병목 분석과 개선
