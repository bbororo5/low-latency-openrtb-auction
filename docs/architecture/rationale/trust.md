# 신뢰 경계 관점 검토

검토 대상: [아키텍처 동인](../drivers.md)

## 결론

공급자·렌더링 클라이언트와 모든 DSP 회사는 SSP의 신뢰 경계 밖에 있다. SSP도 프로젝트 DSP의 신뢰 경계 밖에 있다. 상대를 인증해도 그 내용이 올바르다는 뜻은 아니며, 각자는 자신이 소유한 사실과 연결해 검증한다.

## 사실의 권한

| 주체 | 주장할 수 있는 사실 | 확정할 수 없는 사실 |
|---|---|---|
| 렌더링 클라이언트 | 광고가 렌더링됐다는 주장 | 낙찰가와 캠페인 과금 |
| 외부 DSP | 슬롯에 대한 입찰 | 입찰 유효성·최종 낙찰 |
| SSP | 유효 입찰·낙찰·2초 안의 렌더링과 과금 가능 사건 | 프로젝트 DSP 캠페인 금액 변경 |
| 프로젝트 DSP 입찰 | 캠페인·소재·입찰가와 불투명한 입찰 증표 | SSP의 최종 낙찰 |
| 프로젝트 DSP 예산 | 잠재·확정 지출과 만료 | 렌더링 사실 자체 |

## 경계별 검증

| 경계 | 검증할 것 |
|---|---|
| 공급자 → SSP | 등록된 상대와 요청 권한 |
| 렌더링 클라이언트 → SSP | 실제 낙찰·슬롯 연결, 2초 기한, 중복·변조·재사용 |
| DSP → SSP | 요청·슬롯 연결, 180ms 마감, 가격 정밀도, 통화와 소재 |
| SSP → 프로젝트 DSP | 등록된 SSP, 프로젝트 DSP가 발행한 불투명한 증표와 예약의 연결, 사건 종류, 5초 기한과 중복 |

SSP는 프로젝트 DSP의 캠페인과 내부 예약 정보를 해석하지 않는다. 프로젝트 DSP가 만든 입찰 증표나 통지 주소를 낙찰 근거와 함께 그대로 보존하고 공개 계약 주소로 되돌려 보낸다. 프로젝트 DSP만 그 증표를 해석하고 자기 예약과 금액 불변식을 검증한다.

외부 입력의 가격·캠페인·시각을 금액 원본으로 사용하지 않는다. 실제 사람이 광고를 보았다는 증명과 광고 사기 탐지는 범위 밖이며, 유효한 낙찰에 연결된 클라이언트 통지를 렌더링 성공으로 간주한다. 증표·서명·서버 상태의 구체적인 방식은 아직 선택하지 않는다.

## 결정 결과

- [제품 요구사항](../../requirements/product.md)은 경매·슬롯·예약 식별자와 2초·5초 기한으로 외부 주장을 내부 사실에 연결한다.
- [ADR-004](../decisions/ADR-004-auction-execution-path.md)는 SSP의 인증된 렌더링 증표를 선택하고, [ADR-005](../decisions/ADR-005-durable-budget-events.md)는 DSP의 불투명 통지 URL과 내부 멱등 처리를 선택했다.
- 서버 인증, 증표 형식, 키 관리와 통지 주소 제한은 이 경계를 구현하는 상세 설계로 남긴다.
