# 데이터 중심 시스템 관점 검토

검토 대상: [아키텍처 동인](../drivers.md)

## 결론

핵심 압력은 빠른 조회보다 **데이터별 수명과 일관성의 차이**다. 경매 계산은 버리고 낙찰은 인증된 증표로 운반한다. SSP와 DSP는 자기 지역의 사건을 독립적으로 보존하며 상대 장부의 동시 일치를 요구하지 않는다. 잠재 지출은 무기한 보존하지 않고 DSP 예약 시각부터 5초 안에 끝낸다.

## 데이터 판정

| 데이터 | 성격 | 필요한 성질 |
|---|---|---|
| 캠페인 실행 데이터 | 시험 전에 확정한 입찰용 조회 데이터 | 같은 버전의 일관된 배포와 로컬 조회 |
| 경매 진행 상태 | 180ms 안의 작업 상태 | 유실 시 실패 허용, 늦은 결과 금지 |
| 렌더링 증표 | 낙찰·렌더링 연결 근거 | 위조 방지, 만료와 재사용 판정, 서버 저장 불필요 |
| SSP 청구 사건 | SSP가 검증한 지역 청구 근거 | 지역 내구 append와 비동기 병합 |
| 통지 전달 결과 | SSP가 소유한 HTTP 호출 결과 | SSP 정책에 따라 보존하며 DSP가 응답 본문을 제공할 필요 없음 |
| 잠재·확정 지출 | 프로젝트 DSP의 금액 원본 | 캠페인 총예산 불변식과 5초 만료 |
| 지표·로그 | 검증용 파생 데이터 | 원본을 대신하지 않고 흐름을 연결 |

## 확인된 압력

- 입찰 성공은 캠페인 예산 예약이 복구 가능해진 뒤에만 인정할 수 있다.
- SSP와 DSP의 금액 사건은 인스턴스·AZ 장애 뒤 지역에서 복구 가능해야 하며 다른 리전과 실시간 합의하지 않는다.
- 정확한 총예산과 여러 DSP 인스턴스의 동시 입찰은 강한 쓰기 경합을 만든다.
- 페이싱은 확정 지출뿐 아니라 아직 끝나지 않은 잠재 지출과 상태 가시성 지연의 영향을 받는다.
- `lurl`·`burl`은 중복될 수 있지만 금액 효과는 한 번이어야 한다.
- 입찰 응답이 유실되어도 예약은 5초 안에 끝나야 한다.
- SSP와 DSP는 상대 장부와 메시지 기반 시설을 공유하지 않는다. 외부 계약은 OpenRTB 통지 URL과 일반 HTTP 응답뿐이고 DSP는 재호출을 멱등 처리한다.
- 처리 주체·인스턴스·배치 변경은 이미 만든 잠재 지출을 끝내지 않는다. 사용 여부가 불확실한 금액은 잠재 지출 가능 시간이 끝날 때까지 보수적으로 점유한다.

## 결정 결과

- [ADR-001](../decisions/ADR-001-distributed-budget-reservation.md)~[ADR-003](../decisions/ADR-003-regional-budget-allocation.md)은 전역·리전·DSP 로컬 권한으로 총예산을 보호한다.
- [ADR-005](../decisions/ADR-005-durable-budget-events.md)는 표준 통지, DSP 내부 멱등 기록, 5초 만료와 리스 단위 정산을 선택했다.
- 금액 자료형, 물리 저장소 경계와 페이싱 투영은 현재 아키텍처의 데이터 관점에서 구체화한다.
