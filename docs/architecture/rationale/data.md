# 데이터 중심 시스템 관점 검토

검토 대상: [아키텍처 동인](../drivers.md)

## 결론

핵심 압력은 빠른 조회보다 **데이터별 수명과 일관성의 차이**다. 경매 계산은 버릴 수 있지만 공개한 낙찰, SSP가 성공으로 접수한 과금 가능 사실, DSP가 성공으로 접수한 `burl`과 캠페인 금액은 복구해야 한다. 잠재 지출은 무기한 보존하지 않고 DSP 예약 시각부터 95초 안에 끝낸다.

## 데이터 판정

| 데이터 | 성격 | 필요한 성질 |
|---|---|---|
| 캠페인 실행 데이터 | 시험 전에 확정한 입찰용 조회 데이터 | 같은 버전의 일관된 배포와 로컬 조회 |
| 경매 진행 상태 | 180ms 안의 작업 상태 | 유실 시 실패 허용, 늦은 결과 금지 |
| 공개한 낙찰 결과 | 렌더링·통지 근거 | 장애 뒤에도 결과 불변 |
| 과금 가능 렌더링 접수 | SSP가 검증한 수취 근거 | 최초 접수 시각, 낙찰 연결과 `burl` 발신 책임을 RPO 0으로 복구 |
| 통지 전달 상태 | SSP의 HTTP 재시도 작업 | 멱등 재시도와 기한 안의 접수 판정 |
| 잠재·확정 지출 | 프로젝트 DSP의 금액 원본 | 캠페인 총예산 불변식과 95초 만료 |
| 지표·로그 | 검증용 파생 데이터 | 원본을 대신하지 않고 흐름을 연결 |

## 확인된 압력

- 입찰 성공은 캠페인 예산 예약이 복구 가능해진 뒤에만 인정할 수 있다.
- SSP가 성공으로 접수한 과금 가능 렌더링과 DSP가 성공으로 접수한 `burl`은 장애 뒤 복구 가능한 경계가 필요하다.
- 정확한 총예산과 여러 DSP 인스턴스의 동시 입찰은 강한 쓰기 경합을 만든다.
- 페이싱은 확정 지출뿐 아니라 아직 끝나지 않은 잠재 지출과 상태 가시성 지연의 영향을 받는다.
- `lurl`·`burl`은 중복될 수 있지만 금액 효과는 한 번이어야 한다.
- 입찰 응답이 유실되어도 예약은 95초 안에 끝나야 한다.
- SSP와 DSP는 상대 장부와 메시지 기반 시설을 공유하지 않으며 공개 HTTP 주소의 멱등 재시도로 각자의 상태를 연결한다.
- 처리 주체·인스턴스·배치 변경은 이미 만든 잠재 지출을 끝내지 않는다. 사용 여부가 불확실한 금액은 잠재 지출 가능 시간이 끝날 때까지 보수적으로 점유한다.

## 결정 결과

- [ADR-001](../decisions/ADR-001-distributed-budget-reservation.md)~[ADR-003](../decisions/ADR-003-regional-budget-allocation.md)은 전역·리전·DSP 로컬 권한으로 총예산을 보호한다.
- [ADR-005](../decisions/ADR-005-durable-budget-events.md)는 독립 발신함·수신함, 멱등 반영과 95초 만료를 선택했다.
- 금액 자료형, 물리 저장소 경계와 페이싱 투영은 현재 아키텍처의 데이터 관점에서 구체화한다.
