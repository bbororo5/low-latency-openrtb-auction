# 진화적 아키텍처 관점 검토

검토 대상: [아키텍처 동인](../drivers.md)

## 결론

변화를 미리 모두 추상화하지 않는다. 이미 예상한 통신 표현, 복제본·리전 구성, 캠페인 조회와 금액 저장 방식만 변화 지점으로 관리하고, 경매·금액·기한의 의미는 자동 시험으로 고정한다.

## 유지할 성질

- 180ms 뒤 입찰은 낙찰되지 않는다.
- 슬롯별 1가격 낙찰과 동점 규칙이 바뀌지 않는다.
- CPM과 노출 금액의 정밀도를 잃지 않는다.
- 캠페인 총예산 초과와 중복 과금은 0건이다.
- 모든 잠재 지출은 예약 후 5초 안에 끝난다.
- 인스턴스·AZ 장애에서 지역 금액 사건을 잃지 않고 리전 간 실시간 합의 없이 모순 없는 금액 결과로 수렴한다.
- 정상 첨두 p99 50ms와 페이싱 합격 기준을 반복 판정할 수 있다.

## 예상 변화와 경계

| 변화 | 지킬 경계 |
|---|---|
| HTTP/JSON → 이진 표현 | 경매·가격·통지 의미와 시험 유지 |
| 외부 DSP 추가·교체 | 한 연동의 계약과 실패 격리 |
| 캠페인 조회·페이싱 변경 | 후보 자격, 총예산과 선택 결과 검증 |
| 금액 자료형·저장소 변경 | 기존 예약·과금·만료 의미 보존 |
| 복제본·리전 구성 변경 | 장애 영역과 무관한 금액 효과 |
| 순차 배포 | 이전·새 통신과 저장 상태의 공존 |

## 결정 결과

- 식별자, 가격, 경매와 금액 상태는 [제품 요구사항](../../requirements/product.md)에 고정했다.
- 성능·금액·페이싱·복구 판정은 [품질 요구사항](../../requirements/quality.md)과 [부하·검증 기준](../../requirements/workload.md)에 고정했다.
- 통신·저장 제품은 현재 구조의 의미와 합격 시험을 바꾸지 않는 범위에서 이후 선택한다.
