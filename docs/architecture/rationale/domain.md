# 도메인 경계 관점 검토

검토 대상: [아키텍처 동인](../drivers.md)

## 결론

경매, 렌더링·통지, 캠페인 입찰, 예산은 서로 다른 사실과 불변식을 소유한다. 다만 별도 서비스로 나눠야 한다는 뜻은 아니다. 먼저 논리적 책임을 분리하고 배포·저장 경계는 ADR에서 결정한다.

## 책임 경계

| 경계 | 소유하는 의미 | 소유하지 않는 의미 |
|---|---|---|
| SSP 경매 | 요청 검증, 마감, 슬롯별 1가격 낙찰 | DSP 캠페인과 예산 |
| SSP 렌더링·통지 | 60초 렌더링 판정, 통지 생성·재시도 | 캠페인 금액 변경 |
| 외부 DSP 연동 | 외부 응답 변환과 실패 격리 | 외부 회사 내부 상태 |
| 프로젝트 DSP 입찰 | 후보·페이싱·캠페인·소재·입찰가 | SSP의 최종 낙찰 |
| 프로젝트 DSP 예산 | 잠재·확정 지출, 총예산과 95초 만료 | SSP의 렌더링 사실 |

SSP 회사와 프로젝트 DSP 회사는 독립 배포·독립 저장소를 사용한다. SSP에서 보면 프로젝트 DSP와 두 외부 DSP는 같은 계약을 따르는 별도 회사다. 프로젝트 DSP의 게이트웨이와 여러 인스턴스만 같은 회사의 하나의 논리적 예산을 다룬다.

## 경계를 넘는 사실

| 흐름 | 전달 의미 |
|---|---|
| SSP → DSP | 경매·슬롯·마감·최저가·통화 |
| 프로젝트 DSP 입찰 → 예산 | 캠페인·슬롯·입찰 금액과 예약 요청 |
| DSP → SSP | 요청·슬롯과 연결된 입찰 |
| 클라이언트 → SSP | 렌더링 주장 |
| SSP → DSP | 공개 계약 주소로 보내는 `nurl` 낙찰, `lurl` 패찰, `burl` 과금 가능 사건 |

SSP가 과금 가능 사건을 자기 청구 근거로 기록한 사실과 광고주 캠페인 금액이 확정됐다는 사실은 다르다. 프로젝트 DSP는 예약 후 95초 안에 실제 접수한 `burl`만 캠페인 지출로 확정한다. 두 회사는 공유 이벤트 기반 시설이나 분산 트랜잭션 없이 DSP가 공개한 표준 URL로 통지하며, 통지 유실은 미과금으로 끝날 수 있다. 청구서·송금·매체사 지급과 업체 간 대사는 별도 문제다.

## 결정 결과

- [ADR-001](../decisions/ADR-001-distributed-budget-reservation.md)은 입찰 전에 DSP 로컬 권한에서 잠재 지출을 만든다.
- [ADR-004](../decisions/ADR-004-auction-execution-path.md)는 입찰과 배경 예산 처리의 실행 자원을 분리하고 입찰 경로의 원격 원장 호출을 금지한다.
- [ADR-005](../decisions/ADR-005-durable-budget-events.md)는 SSP와 DSP가 저장소와 메시지 기반 시설을 공유하지 않고 각자 인정한 과금 사실을 보존하도록 정했다.
- 캠페인 변경 전파와 내부 모듈 경계는 현재 아키텍처와 상세 설계에서 구체화한다.
