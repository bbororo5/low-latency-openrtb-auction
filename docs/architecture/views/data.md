# 데이터 접근·보존 기준

상태: 예산 원장 경계 확정·캠페인 경계 검토 중

근거: [제품·도메인 요구사항](../../requirements/product.md), [부하·데이터·검증 기준](../../requirements/workload.md), [아키텍처 중요 요구사항](../../requirements/quality.md), [ADR-001~006](../decisions/)

이 문서는 제품을 선택하지 않고 데이터별 접근 경로와 보존 수준을 정한다. 확정한 물리 저장소 경계와 아직 검토할 경계를 함께 관리한다.

## 1. 접근·보존 기준

| 데이터 | 접근과 규모 | 지연 경로 | 일관성·복구 | 자료구조와 저장 경계에 주는 요구 |
|---|---|---|---|---|
| SSP 경매 작업 상태 | 요청당 생성·종결, 최대 180ms | 경매 Hot Path | 유실 시 해당 요청 실패 허용 | 인스턴스 메모리, 요청 종료 후 폐기 |
| SSP 낙찰·렌더링·통지 발신함 | 성공 경매와 렌더링마다 기록 | 경매 성공 응답 및 비동기 통지 | 성공 사실 RPO 0, 사건별 멱등성 | 성공 응답 전 내구 기록, 낙찰과 발신함의 원자적 생성 필요 |
| 캠페인 원본 | 총 10만, 활성 3만, 실행 중 변경 적음 | 입찰 경로 밖에서 배포 | 버전 보존, 로컬 사본은 재생성 가능 | 내구 원본과 버전 단위 배포 |
| 캠페인 실행 스냅숏 | 요청당 후보 p99 1,000개 조회 | 입찰 Hot Path | 버전 단위 일관성, 종료 시각은 로컬 판정, 중지 상태는 전파 지연 상한 필요 | 불변 배열·해시 인덱스·규격별 후보 인덱스, 참조 원자 교체 |
| 전역 예산 책임 | 캠페인별 총예산·예비액·리전 이전, 저빈도 쓰기 | 입찰 경로 밖 | 책임 이전과 인정된 금액 사실 RPO 0, 중복 이전 금지 | 캠페인별 조건부 원자 갱신과 멱등 이전 기록 |
| 리전 예산 책임 | DSP 권한 발급·격리·회수, 인기 캠페인 편향 | 입찰 경로 밖의 리전 제어 | 리전 책임액 초과 금지, 리전 내부 고가용, 세부 기록은 비동기 복구 허용 | 캠페인·인스턴스·권한 세대별 레코드와 단일 작성 권한 |
| DSP 로컬 예산 권한 | 입찰마다 원자 차감 | 입찰 Hot Path | 인스턴스 유실 허용, 상위 권한을 격리해 복구 | 캠페인별 정수 카운터와 권한 세대, 외부 조회 금지 |
| 잠재 지출 | 슬롯별 생성, `lurl`·`burl`·95초 만료로 종결 | 생성은 입찰 Hot Path, 종결은 통지 경로 | 개별 로컬 기록 유실 허용, 불확실한 상위 권한은 보수적으로 점유 | 예약 ID 해시 조회와 만료 버킷·타이밍 휠, 비동기 누적 보고 |
| DSP 경매 결과 사건 | `nurl` 중복 접수 | 경매와 분리된 통지 경로 | 금액 효과 없음, 유실 허용 | 추적·분석이 필요하면 사건 ID로 중복을 구분하되 금액 원본과 분리 |
| DSP 외부 금액 사건 | `lurl`·`burl` 중복 접수·재처리 | 경매와 분리된 통지 경로 | 접수 성공 전 RPO 0, 사건 ID별 금액 효과 한 번 | 고유 사건 ID가 있는 내구 수신함, 원장 반영 상태 보존 |
| 페이싱 투영 | 예약·해제 누계를 주기적으로 집계하고 요청마다 조회 | 집계는 비동기, 조회는 입찰 Hot Path | 최종 일관성, 원본에서 재생성 가능 | 권한 세대별 누적 카운터, 규격별 후보 순서가 있는 불변 로컬 스냅숏 |

## 2. 데이터 등급

| 등급 | 포함 데이터 | 원칙 |
|---|---|---|
| 로컬 실행 데이터 | 경매 작업 상태, 캠페인 스냅숏, DSP 권한, 잠재 지출 | 입찰 경로에서 네트워크와 외부 저장소를 사용하지 않는다. |
| 강하게 보존할 사실 | SSP 성공 사실, 전역 예산 책임, 성공 접수한 금액 사건 | 성공을 반환하기 전에 리전 하나의 장애 뒤에도 복구 가능해야 한다. |
| 리전 책임 상태 | 리전 책임액, DSP 권한 발급·격리·회수 | 리전 안에서 원자적으로 제한하고 반대 리전에는 복구용으로 늦게 복제할 수 있다. |
| 운영·파생 데이터 | `nurl`, 후보 인덱스, 페이싱 투영, 지표 | 금액 원본이 아니며 지연과 유실을 허용한다. 재생성 가능한 자료는 원본에서 다시 만든다. |

## 3. 확정된 금액 사건 경계

DSP는 유효한 `lurl`·`burl`을 내구 수신함에 RPO 0으로 기록한 뒤 성공을 반환한다. 원장 처리는 수신함을 입력으로 비동기 수행하며 사건 ID와 예약 ID로 금액 효과를 한 번만 반영한다. 따라서 원장 지연이나 일시 장애가 이미 성공으로 접수한 사건을 잃게 만들지 않는다.

수신함과 원장이 같은 저장 제품을 사용할 수는 있지만, 성공 접수를 원장 반영과 하나의 동기 트랜잭션으로 묶지 않는다.

## 4. 확정된 예산 원장 경계

전역 책임 원장과 각 리전 예산 원장은 별도 물리 저장소 배포 경계로 분리한다. 같은 저장 기술은 사용할 수 있지만 클러스터·장애·합의 범위는 공유하지 않는다.

- 전역 책임 원장: 두 리전에 강하게 보존하며 전역 예비액과 리전 책임 이전을 소유한다.
- 리전 예산 원장: 해당 리전 안에서 고가용으로 운영하며 자기 책임액과 DSP 권한 발급·격리·회수를 소유한다.
- 전역 원장이 중단되어도 각 리전은 기존 책임액으로 계속 처리하고 새 책임 이전만 중단한다.

## 5. 아직 결정하지 않은 경계

1. 캠페인 원본의 배포 방식과 페이싱 집계 상태를 어디에 둘지

이 항목을 결정하기 전에는 캠페인·페이싱 저장소의 물리 경계를 확정하지 않는다.
