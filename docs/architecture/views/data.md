# 데이터 접근·보존 기준

상태: 데이터 저장 경계와 지역 금액 장부·전역 책임 원장 기술 확정

근거: [제품·도메인 요구사항](../../requirements/product.md), [부하·데이터·검증 기준](../../requirements/workload.md), [아키텍처 중요 요구사항](../../requirements/quality.md), [ADR-001~008](../decisions/)

이 문서는 데이터별 접근 경로와 보존 수준을 정하고 확정한 저장 기술과 아직 검토할 경계를 함께 관리한다.

## 1. 접근·보존 기준

| 데이터 | 접근과 규모 | 지연 경로 | 일관성·복구 | 자료구조와 저장 경계에 주는 요구 |
|---|---|---|---|---|
| SSP 경매 작업 상태 | 요청당 생성·종결, 최대 180ms | 경매 Hot Path | 유실 시 해당 요청 실패 허용 | 인스턴스 메모리, 요청 종료 후 폐기 |
| SSP 경매 결과 | 성공 경매마다 생성 | 경매 성공 응답과 렌더링 검증 | 서버 상태를 보존하지 않고 인증된 렌더링 증표로 전달 | 요청 메모리와 위조 방지 증표, 외부 저장소 쓰기 없음 |
| SSP 금액 사건 | 유효한 렌더링마다 추가 | 렌더링 접수와 HTTP 통지 | 리전 내부 내구성, 사건별 멱등성, 리전 간 비동기 병합 | `BillingClaimRecorded` 자체가 `burl` 전달 책임이며 투영은 재생성 가능 |
| 캠페인 원본 | 총 10만, 활성 3만, 시험 중 불변 | 시작 전에 배포 | 버전 보존, 로컬 사본은 재생성 가능 | 내구성 있는 버전 자료와 체크섬 |
| 캠페인 실행 스냅숏 | 요청당 후보 p99 1,000개 조회 | 입찰 Hot Path | 시험 동안 같은 버전 사용, 시작·종료 시각과 활성 상태는 로컬 판정 | 불변 배열·해시 인덱스·규격별 후보 인덱스, 시작 전 적재 |
| 전역 예산 책임 | 캠페인별 총예산·예비액·리전 이전, 저빈도 쓰기 | 입찰 경로 밖 | 중복 이전 금지, 홈 리전 유실 시 자동 승격 금지와 보수적 동결 | PostgreSQL 조건부 원자 갱신, 멱등 이전 기록과 쓰기 세대 |
| 리전 예산 책임 | DSP 권한 발급·격리·회수, 인기 캠페인 편향 | 입찰 경로 밖의 리전 제어 | 리전 책임액 초과 금지, 리전 내부 고가용, 세부 기록은 비동기 복구 허용 | 캠페인·인스턴스·권한 세대별 레코드와 단일 작성 권한 |
| DSP 로컬 예산 권한 | 입찰마다 원자 차감 | 입찰 Hot Path | 인스턴스 유실 허용, 상위 권한을 격리해 복구 | 캠페인별 정수 카운터와 권한 세대, 외부 조회 금지 |
| 잠재 지출 | 슬롯별 생성, `lurl`·`burl`·95초 만료로 종결 | 생성은 입찰 Hot Path, 종결은 통지 경로 | 개별 로컬 기록 유실 허용, 불확실한 상위 권한은 보수적으로 점유 | 예약 ID 해시 조회와 만료 버킷·타이밍 휠, 비동기 누적 보고 |
| DSP 경매 결과 사건 | `nurl` 중복 접수 | 경매와 분리된 통지 경로 | 금액 효과 없음, 유실 허용 | 추적·분석이 필요하면 사건 ID로 중복을 구분하되 금액 원본과 분리 |
| DSP 외부 금액 사건 | `lurl`·`burl` HTTP 중복 접수 | 경매와 분리된 통지 경로 | 리전 내부 내구성, 내부 사건 ID별 금액 효과 한 번, 응답 본문 없음 | PostgreSQL 지역 추가 전용 기록과 리스별 집계 |
| 리스 정산 | 리스마다 한 번 생성 | 입찰 경로 밖 | `leaseId`·정산 세대별 멱등성, 사건 원본에서 재생성 가능 | 확정 소비·미사용 반환·격리 합계와 지역 원장 정산 기록 |
| 페이싱 투영 | 리전 책임액의 순발급 리스와 DSP 로컬 잔여 권한 | 발급은 리전 제어 경로, 조회는 입찰 Hot Path | 리전 안의 결과적 일관성, 유실 시 권한 상태에서 보수적으로 복구 | 리전 원장의 리스 상태와 DSP 로컬 메모리, 리전 간 실시간 합산 없음 |

## 2. 데이터 등급

| 등급 | 포함 데이터 | 원칙 |
|---|---|---|
| 로컬 실행 데이터 | 경매 작업 상태, 캠페인 스냅숏, DSP 권한, 잠재 지출 | 입찰 경로에서 네트워크와 외부 저장소를 사용하지 않는다. |
| 지역 금액 사건 | SSP 청구 근거, DSP가 접수한 `burl` | 성공을 반환하기 전에 리전 내부에서 내구 기록하고 다른 리전과 비동기로 병합한다. |
| 강하게 보존할 책임 | 전역 예산 책임과 리전 책임 이전 | 단계별 원자 기록과 중간 격리로 같은 예산을 두 리전에 동시에 활성화하지 않는다. |
| 리전 책임 상태 | 리전 책임액, DSP 권한 발급·격리·회수 | 리전 안에서 원자적으로 제한하고 반대 리전에는 복구용으로 늦게 복제할 수 있다. |
| 운영·파생 데이터 | `nurl`, 후보 인덱스, 페이싱 투영, 지표 | 금액 원본이 아니며 지연과 유실을 허용한다. 재생성 가능한 자료는 원본에서 다시 만든다. |

## 3. 확정된 금액 사건 경계

DSP는 `burl`의 최초 접수 또는 만료 판정을 PostgreSQL 지역 추가 전용 기록에 남긴 뒤 본문 없는 일반 HTTP 성공을 반환한다. 같은 URL의 재호출은 금액을 다시 바꾸지 않으며 위조·훼손된 URL만 오류로 거부한다. 입찰을 처리한 DSP 리전이 사건을 소유하며 다른 리전은 장애 대체 결정을 만들지 않는다. `lurl` 유실과 오지 않은 `burl`은 잠재 지출의 95초 만료가 안전하게 종결한다.

SSP 청구 근거와 DSP 접수 근거는 독립 원본이며 외부 계약으로 하나의 회계 결정을 만들지 않는다. SSP는 일반 HTTP 호출 결과만 알고 DSP 내부 판정을 해석하지 않는다. 통지 유실로 양사의 자료가 달라질 수 있으며 업체 간 대사는 구현 범위 밖이다.

지역 사건 기록은 PostgreSQL을 사용한다. 리전 예산 원장의 저장 제품은 아직 정하지 않았으며 접수 사건마다 원장을 갱신하지 않는다. 모든 잠재 지출 기한이 끝난 뒤 사건을 리스별로 집계하고 확정 소비와 미사용 반환액을 하나의 멱등 정산으로 반영한다.

## 4. 확정된 예산 원장 경계

전역 책임 원장과 각 리전 예산 원장은 별도 물리 저장소 배포 경계로 분리한다. 같은 저장 기술은 사용할 수 있지만 클러스터·장애·합의 범위는 공유하지 않는다.

- 전역 책임 원장: 홈 리전 RDS PostgreSQL Multi-AZ가 전역 예비액과 리전 책임 이전의 단일 쓰기 권위이며 다른 리전에는 비동기 복구 사본을 둔다.
- 리전 예산 원장: 해당 리전 안에서 고가용으로 운영하며 자기 책임액과 DSP 권한 발급·격리·회수를 소유한다.
- 전역 원장이 중단되어도 각 리전은 기존 책임액으로 계속 처리하고 새 책임 이전만 중단한다. 복구 사본은 지역 활성화 증거를 대조하기 전에 자동 승격하지 않는다.

## 5. 확정된 캠페인·페이싱 경계

캠페인은 시험 전에 버전 자료로 만들고 두 리전의 DSP 인스턴스가 시작할 때 같은 버전과 체크섬을 확인해 로컬 메모리에 적재한다. 시험 중 값이 바뀌지 않으므로 온라인 캠페인 제어 저장소와 변경 전파 경로를 두지 않는다.

페이싱은 별도 저장소나 서비스 없이 리전 예산 원장의 리스 발급과 DSP 로컬 권한으로 처리한다.

```text
리전 순발급 권한 = 누적 리스 발급액 - 증명된 미사용 반환·만료액

리전 순발급 권한
<= 리전 캠페인 책임액 × 기간 경과율 + 허용 편차
```

- 각 리전은 겹치지 않는 자기 책임액만 페이싱하므로 다른 리전의 개별 리스 상태를 합산하지 않는다.
- DSP는 유효한 로컬 권한이 있는 캠페인만 입찰하고, 로컬 리스 사용 상태로 뒤처진 후보를 우선한다.
- 책임액을 다른 리전에 이전할 때는 미사용임을 증명한 금액만 옮기고 남은 기간에 걸쳐 새 리스를 발급한다.
- 리스 상태가 불확실하면 사용된 것으로 간주한다. 이는 예산 초과와 과도한 선행 소비 대신 일시적인 미소진을 선택한다.
- 리스 크기와 유효 기간은 페이싱 편차와 리전 원장 통신량 사이의 조정값이며 시험으로 정한다.

리전별 누적값은 검증 지표로 수집할 수 있지만 런타임 페이싱 제어에는 전역 합산 결과를 사용하지 않는다.
