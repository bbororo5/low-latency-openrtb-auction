## 1. 핵심 동인

| ID | 동인 | 근거 |
|---|---|---|
| `동인-01` | 제한 시간 안에 정확하고 결정적인 경매 결과를 내야 한다. | `ASR-01`, `ASR-02` |
| `동인-02` | 동시 입찰과 중복·역순 사건에서도 예산 불변식을 지켜야 한다. | `ASR-04` |
| `동인-03` | 인스턴스와 AZ 장애에도 서비스를 지속하며 인정된 금액 효과를 잃지 않아야 한다. | `ASR-05` |
| `동인-04` | 외부 DSP 실패와 처리 능력을 넘는 부하가 전체 장애와 장기 지연으로 번지지 않아야 한다. | `ASR-02`, `ASR-03` |
| `동인-05` | 수명·읽기와 쓰기 비율·정합성 요구가 다른 데이터를 각 성격에 맞게 다뤄야 한다. | `ASR-01`, `ASR-04`, `ASR-05` |
| `동인-06` | 분산된 경매와 금액 흐름을 재현·대조할 수 있고 통신 표현을 독립적으로 바꿀 수 있어야 한다. | `ASR-06`, `ASR-07` |

## 2. 데이터 상태와 불변식

| 상태 | 성격 | 보존해야 할 것 |
|---|---|---|
| 경매 진행과 결과 | 제한 시간에 민감하고 대부분 수명이 짧다. | 마감 전 유효 입찰과 결정된 낙찰 결과, 과금 통지와 연결할 식별 관계 |
| 캠페인 실행 데이터 | 원본 시스템에서 공급되는 조회 중심의 파생 데이터다. | 같은 입력과 데이터 버전에서 같은 후보와 입찰 결과 |
| 잠재·확정 지출 | 동시 변경이 집중되는 자사 DSP의 금액 원본 상태다. | 광고주·캠페인 한도와 한 번의 금액 효과 |
| 낙찰·패찰·과금·만료 | 중복·누락·역순·충돌할 수 있는 상태 전이 입력이다. | 사건 식별, 유효성 판단, 최종 금액 상태 |
| 관측 자료 | 경매와 금액의 원본은 아니지만 검증에 필요하다. | 요청부터 낙찰·과금·장애·복구까지의 연결 관계 |

다음 불변식은 저장 방식이나 장애 여부와 관계없이 유지되어야 한다.

```text
광고주 확정 지출 + 잠재 지출 <= 충전액
캠페인 누적 확정 지출 + 잠재 지출 <= 누적 한도
캠페인 일일 확정 지출 + 잠재 지출 <= 일일 한도
```

- 같은 금액 사건은 한 번만 효과를 낸다.
- 과금은 잠재 지출을 확정 지출로 바꾸며 사용 가능 예산을 다시 줄이지 않는다.
- 늦은 입찰은 경매 결과를 바꾸지 않는다.
- 장애 전에 인정한 금액 효과는 유실되거나 중복되지 않는다.

## 3. 책임 경계와 변화 방향

| 책임 경계 후보 | 소유하는 의미 | 독립적으로 변할 가능성이 큰 부분 |
|---|---|---|
| 경매 | 요청 검증, DSP별 결과, 마감, 낙찰 | DSP 수와 연동 규격, 경매 정책, 제한 시간 |
| 캠페인 입찰 | 실행 데이터, 후보 검색, 캠페인·소재 선택, 입찰가 | 원본 데이터 구조, 검색 기준, 입찰 정책 |
| 예산 | 잠재 지출, 확정 지출, 사용 가능 예산 | 금액 정밀도, 동시성 제어, 한도 정책 |
| 과금 연결 | 클라이언트 노출 통지와 경매·입찰의 연결 | 과금 인정 조건, 중복·역순·만료 규칙 |
| 검증 자료 | 성능·장애·금액 대조에 필요한 연결 정보 | 통신 표현, 배포 구조, 관측 도구 |

이 표는 논리적 책임을 구분할 뿐 서비스, 프로세스, 저장소의 분리를 뜻하지 않는다.

현재 확인된 변화 방향은 다음과 같다.

- SSP와 DSP 사이의 JSON 표현은 이진 표현으로 바뀔 수 있다.
- 외부 DSP의 수와 지연·오류 특성은 달라질 수 있다.
- 캠페인 실행 데이터의 공급 방식과 검색 구조는 달라질 수 있다.
- 금액 정밀도와 내부 표현 방식은 별도로 결정해야 한다.
- 과금 인정 조건과 사건 전달 방식은 달라질 수 있다.
- 부하와 복제본 수가 늘어도 도메인 규칙과 합격 시험은 유지되어야 한다.

## 4. 충돌과 우선순위

1. 잘못된 낙찰과 금액 위반을 막는 것이 처리량보다 우선한다.
2. 금액 상태를 보존하기 위해서도 경매 제한 시간을 넘길 수는 없다.
3. 서비스 지속성을 높이는 결정이 인정된 금액 효과의 RPO 0을 약화해서는 안 된다.
4. 과부하 시 모든 요청을 받는 것보다 정확성을 지키며 회복하는 것이 우선한다.
5. 성능 최적화와 통신 변경은 도메인 의미와 검증 기준을 바꾸지 않는 범위에서만 허용한다.

## 5. ADR로 넘길 결정 질문

- 경매의 남은 시간을 세 DSP 호출에 어떻게 적용하고 늦은 작업을 어떻게 끝낼 것인가?
- 외부 실패와 순간 부하가 자원 고갈과 장기 지연으로 번지는 것을 어떻게 막을 것인가?
- 캠페인 실행 데이터를 어떤 형태로 공급·보관·조회할 것인가?
- CPM과 예산을 어떤 정밀도로 표현하고 동시 지출을 어떻게 조정할 것인가?
- 낙찰·패찰·과금·만료의 중복·역순·충돌을 어떤 상태 전이로 판정할 것인가?
- 과금 연결 정보와 금액 상태를 어디에 보존하고 AZ 장애에서 어떻게 복구할 것인가?
- 경매와 금액 의미를 통신 표현 및 관측 도구로부터 어떻게 분리할 것인가?

이 질문의 대안과 트레이드오프를 ADR에서 결정한다.
