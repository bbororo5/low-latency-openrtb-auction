| ID | 동인 | 근거 |
|---|---|---|
| `동인-01` | 제한 시간 안에 정확하고 결정적인 경매 결과를 내야 한다. | `ASR-01`, `ASR-02` |
| `동인-02` | 동시 입찰과 중복·역순 사건에서도 예산 불변식과 한 번의 금액 효과를 지켜야 한다. | `ASR-04` |
| `동인-03` | 인스턴스와 AZ 장애에도 서비스를 지속하며 성공으로 인정한 금액 효과를 잃지 않아야 한다. | `ASR-05` |
| `동인-04` | 외부 지연, 내부 자원 고갈, 재시도와 순간 부하가 전체 장애와 장기 지연으로 번지지 않아야 한다. | `ASR-02`, `ASR-03` |

## 2. 데이터 상태와 불변식

| 상태 | 성격 | 보존 기준 |
|---|---|---|
| 경매 진행 상태 | 제한 시간 안에서만 유효한 작업 상태 | 유실 시 해당 경매는 실패할 수 있지만 늦은 결과를 만들 수 없다. |
| 낙찰과 과금 연결 정보 | 과금 종료까지 필요한 근거 | 잠재 지출 만료 전까지 원래 가격·통화·입찰과 연결할 수 있어야 한다. |
| 캠페인 실행 데이터 | 원본 시스템에서 공급되는 조회용 파생 데이터 | 데이터 버전별로 결정적이며 재구축할 수 있어야 한다. |
| 잠재·확정 지출 | 자사 DSP가 소유하는 금액 원본 | 동시 변경과 장애에서도 광고주·캠페인 한도를 함께 지켜야 한다. |
| 낙찰·패찰·과금·만료 | 중복·누락·역순·충돌할 수 있는 상태 전이 입력 | 사건 식별과 최종 상태를 복구·대조할 수 있어야 한다. |
| 지표와 로그 | 검증용 파생 데이터 | 금액 원본을 대신하지 않으며 경계 간 흐름을 추적할 수 있어야 한다. |

```text
광고주 확정 지출 + 잠재 지출 <= 충전액
캠페인 누적 확정 지출 + 잠재 지출 <= 누적 한도
캠페인 일일 확정 지출 + 잠재 지출 <= 일일 한도
```

- 입찰 응답으로 사용된 잠재 지출과 성공으로 응답한 금액 사건은 장애 뒤에도 복구할 수 있어야 한다.
- 같은 금액 사건은 여러 번 전달될 수 있지만 금액 효과는 한 번만 발생한다.
- 과금은 잠재 지출을 확정 지출로 바꾸며 사용 가능 예산을 다시 줄이지 않는다.
- 과금이 늦게 도착해도 경매 수신 시각의 날짜와 원래 낙찰가를 사용한다.
- 늦은 입찰은 경매 결과를 바꾸지 않는다.

## 3. 책임 경계

| 경계 | 소유하는 의미 |
|---|---|
| SSP 경매 | 요청 검증, 외부·자사 DSP 응답 판정, 마감, 낙찰, 공급자 통지와 낙찰의 연결 |
| 외부 DSP 연동 | 외부 회사의 입찰 응답을 내부 경매 결과 후보로 변환하는 규칙 |
| 자사 DSP 입찰 | 실행 데이터, 후보 검색, 캠페인·소재 선택, 입찰가 |
| 자사 DSP 예산 | 잠재 지출, 확정 지출, 사용 가능 예산과 금액 상태 전이 |

과금은 하나의 독립 경계가 아니라 공급자 통지를 SSP가 낙찰과 연결하고 자사 DSP 예산이 금액 상태를 바꾸는 경계 간 흐름이다. 검증 자료는 도메인 경계가 아니라 모든 경계를 가로지르는 운영 관심사다.

이 구분은 코드와 모델의 소유권을 뜻하며 서비스, 프로세스, 저장소의 분리를 뜻하지 않는다.

## 4. 변화·실패·신뢰 원칙

- JSON에서 이진 표현으로 바꾸거나 복제본을 늘려도 경매와 금액 의미는 바뀌지 않는다.
- 순차 배포 중 이전 버전과 새 버전의 통신·사건·저장 상태가 함께 동작해야 한다.
- 외부 DSP 하나의 실패는 해당 입찰만 제외하며 다른 DSP와 경매 전체로 퍼지지 않는다.
- 처리 능력을 넘으면 제한된 요청을 실패시키되 잘못된 낙찰과 금액 위반을 만들지 않는다.
- 클라이언트는 렌더링을 주장할 뿐 과금을 확정하지 못한다.
- 공급자 통지의 가격은 SSP가 보존한 낙찰 결과와 대조하며, 자사 DSP만 예산을 변경한다.
- 송신자를 확인한 뒤에도 식별자, 권한, 현재 상태, 중복과 허용 시간을 검증한다.

## 5. 충돌과 우선순위

1. 잘못된 낙찰과 금액 위반을 막는 것이 처리량보다 우선한다.
2. 금액 상태를 보존하기 위해서도 경매 제한 시간을 넘길 수는 없다.
3. 금액 저장 결과가 불확실하면 성공으로 응답하거나 입찰에 사용하지 않는다.
4. 가용성을 높이는 결정이 성공으로 인정한 금액 효과의 RPO 0을 약화해서는 안 된다.
5. 성능 최적화와 통신 변경은 도메인 의미와 자동 검증 기준을 바꾸지 않는 범위에서만 허용한다.

## 6. ADR로 넘길 결정 질문

- 경매 시간 예산과 외부 호출, 자원 상한, 과부하 거절을 어떻게 제어할 것인가?
- 캠페인 실행 데이터를 어떤 계약과 버전으로 공급·보관·조회할 것인가?
- CPM과 예산을 어떤 정밀도로 표현하고 광고주·캠페인 동시 지출을 어떻게 조정할 것인가?
- 낙찰·패찰·과금·만료의 식별, 성공 인정, 중복·역순·충돌을 어떤 상태 전이로 판정할 것인가?
- 낙찰 연결 정보와 금액 상태를 어디에 보존하고 인스턴스·AZ 장애에서 어떻게 복구할 것인가?
- 클라이언트, 공급자, 외부 DSP, SSP, 자사 DSP 사이의 신원·권한·내용·재전송을 어떻게 검증할 것인가?
- 순차 배포 중 통신·사건·저장 계약의 호환성을 어떻게 유지할 것인가?
- 경매·금액 의미를 통신과 관측 도구에서 어떻게 분리하고 합격 기준을 자동 판정할 것인가?

각 질문의 대안과 트레이드오프를 ADR에서 결정한다.
