# ADR-001 분산 캠페인 예산 예약

상태: 잠정 결정

근거: [아키텍처 중요 요구사항](../asr.md), [아키텍처 동인](../architecture-drivers.md)

## 맥락

여러 AZ의 자사 DSP 인스턴스가 같은 캠페인 예산으로 동시에 입찰한다. 초과 지출·중복 과금 0건과 인정한 금액 사실의 RPO 0을 지키면서 정상 500 RPS와 순간 1,000 RPS에서 정상·보호 요청의 p99 50ms를 만족해야 한다.

과금 정합성과 받아들인 정상·보호 요청의 p99 50ms는 함께 통과해야 하는 절대 조건이다. 둘 중 하나를 완화해 동시성이나 장애 시 가용성을 얻지 않는다.

| 우선순위 | 아키텍처 특성 | 충돌 시 원칙 |
|---|---|---|
| 최우선 | 과금 정합성, 레이턴시 | 둘 다 보호한다. 하나라도 확신할 수 없는 요청은 처리하지 않는다. |
| 후순위 | 동시성 | 초과 요청과 자사 DSP 입찰 기회를 줄여 앞선 두 특성을 보호한다. |
| 후순위 | 장애 시 보호 | 불확실한 상태로 빠르게 재개하지 않고 안전하게 중단한 뒤 복구한다. 복구가 느려질 수 있다. |

동시성과 장애 회복력은 중요하지만 앞선 두 특성을 침해하지 않는 범위에서 높인다. 후순위라는 말은 400 RPS와 복구 목표를 삭제한다는 뜻이 아니라, 목표가 충돌하거나 범위를 넘는 상황에서 무엇을 희생할지 정한 것이다.

## 결정

캠페인 예산, 잠재 지출과 확정 지출을 소유하는 중앙 권위 저장소에서 입찰마다 원자적으로 예산을 예약한다.

자사 DSP는 캠페인과 금액을 결정한 뒤 고유한 논리 예약 식별자로 중앙 예약을 요청한다. 중앙 저장소가 다음 불변식을 만족하는 예약을 복구 가능하게 보존한 경우에만 입찰 응답을 만든다.

```text
캠페인 확정 지출 + 유효한 잠재 지출 <= 캠페인 총예산
```

논리 예약 식별자는 경매와 슬롯으로 정한다. 캠페인을 식별자에 넣지 않아 재시도가 다른 캠페인을 골라도 같은 슬롯의 예약을 두 번 만들지 못하게 한다.

한 경매의 여러 슬롯은 중앙 저장소에 한 번의 원격 요청으로 전달한다. 예약과 결과는 슬롯별로 독립적이므로 한 슬롯의 예산이 부족해도 다른 슬롯의 유효한 예약은 성공할 수 있다. 같은 캠페인을 사용하는 슬롯은 고정된 슬롯 순서로 조건부 예약하며 성공한 금액의 합이 캠페인 한도를 넘지 않도록 한 번의 중앙 처리에서 보장한다. 같은 논리 예약의 재시도는 새 잠재 지출을 만들지 않고 기존 결과를 반환한다.

예약 뒤 최초로 인정한 유효 사건이 금액 상태를 한 번만 바꾼다.

- `lurl`: 잠재 지출 해제
- `burl`: 확정 지출로 전환
- 예약 후 95초 동안 유효한 사건 없음: 잠재 지출 해제

중복 사건은 기존 결과를 반환하고 모순되거나 늦은 사건은 금액을 다시 바꾸지 않는다. 성공 응답은 해당 상태를 장애 뒤에도 복구할 수 있다는 뜻이다.

DSP의 로컬 메모리와 캐시는 예산 원본으로 사용하지 않는다. 중앙 예약 결과를 확신할 수 없거나 제한 시간 안에 완료할 수 없으면 새 자사 DSP 입찰을 만들지 않는다. SSP는 가능한 경우 외부 DSP와 경매를 계속한다.

중앙 저장소의 제품, 복제와 다중 AZ 배치는 별도 결정으로 다룬다. 이 ADR은 애플리케이션이 자체 합의 원장을 구현하지 않고, 원자적 갱신과 RPO 0을 제공하는 권위 저장소를 사용한다는 범위까지만 정한다.

## 대안 도출

출발점은 제품이나 업계 사례가 아니라 예산 불변식이다. 서로 통신하지 않는 두 DSP가 분할되지 않은 같은 잔액을 보고 동시에 예약하면 둘 다 성공할 수 있으므로 초과 지출 0건을 보장할 수 없다. 따라서 예약 전이나 예약 시점에 소비 권한을 조정해야 한다.

대안은 **조정하는 시점과 장소**를 기준으로 도출했다.

| 조정 시점 | 조정 장소 | 도출되는 대안 | 기술적 계보 |
|---|---|---|---|
| 매 예약 | 공유 권위 저장소 | A. 중앙 원자적 예약 | 직렬화 가능한 트랜잭션, 원자적 조건부 갱신 |
| 매 예약 | 캠페인별 활성 소유자 | B. 캠페인별 단일 소유자 | 주 복제본, 단일 작성자, 리더 기반 복제 상태 기계 |
| 예약 전 | DSP별로 분할한 소비 권한 | C. 예산 권한 선할당 | 에스크로 트랜잭션, 할당량 분할 |
| 예약 후 | 중앙에서 사후 병합 | D. 독립 예약 후 비동기 조정 | 낙관적 실행, 최종 일관성과 충돌 조정 |

이 네 가지는 RTB에만 존재하는 제품 목록이 아니라 분산된 한도 차감 문제의 대표 설계 계열이다. 합의 정족수, 샤딩, 특정 저장소, 배치와 캐시는 각 계열의 구현 방식이거나 혼합형이므로 별도 대안으로 세지 않았다.

A의 저장소도 내부적으로 리더와 합의를 사용할 수 있다. A와 B의 차이는 합의 알고리즘이 아니라 예약 직렬화와 장애 전환을 저장소에 맡기는지, 애플리케이션이 캠페인 소유권으로 직접 책임지는지다. D는 요구사항을 만족할 후보라기보다 조정을 제거했을 때 얻는 이점과 잃는 안전성을 확인하는 경계 대안이다.

이 분류의 근거는 다음 연구와 연결된다.

- 조정 없이 병합한 상태가 불변식을 깨면 정확성을 위해 조정이 필요하다는 [불변식 합류성 연구](https://www.vldb.org/pvldb/vol8/p185-bailis.pdf)
- 수량 일부를 미리 떼어 동시 소비하는 [에스크로 트랜잭션](https://doi.org/10.1145/7239.7265)
- 단일 리더가 명령 순서와 복제를 책임지는 [Raft 복제 상태 기계](https://raft.github.io/raft.pdf)
- 가용성을 위해 일부 장애 상황의 일관성을 포기하고 충돌을 사후 조정하는 [Dynamo](https://www.amazon.science/publications/dynamo-amazons-highly-available-key-value-store)

## 대안과 트레이드오프

각 대안은 정합성을 지키는 데 필요한 장애 복구까지 포함하여 비교했다. 단, 금액 안전성을 구조적으로 만족하지 못하는 D는 그 지점에서 평가를 중단했다.

### 구조 비교

| 대안 | 예산을 직렬화하는 곳 | 정상 예약 경로 | 장애 시 핵심 처리 |
|---|---|---|---|
| **A. 중앙 원자적 예약** | 중앙 권위 저장소의 캠페인 상태 | 매 입찰마다 중앙의 복구 가능한 원자적 쓰기 | 중앙의 잠재 지출로 복구하고, 중앙이 불확실하면 자사 입찰 중단 |
| **B. 캠페인별 단일 소유자** | 캠페인의 활성 소유자 하나 | 매 입찰마다 소유자 호출과 대기 복제 | 새 소유자를 선출하고 과거 소유자를 차단한 뒤 재개 |
| **C. 예산 권한 선할당** | 중앙의 권한 발급과 각 DSP의 할당 범위 | 할당 범위에서는 로컬 예약, 부족할 때 중앙 보충 | 권한 만료·회수와 늦은 과금의 업체 손실 처리 |
| **D. 독립 예약 후 비동기 조정** | 공통 직렬화 지점 없음 | 각 DSP가 로컬 예약 후 사후 병합 | 중복 소비를 사후 발견하고 보상 |

### 특성 비교

| 평가축 | A. 중앙 예약 | B. 단일 소유자 | C. 권한 선할당 | D. 비동기 조정 |
|---|---|---|---|---|
| **과금 정합성** | 중앙 원자성으로 초과 지출·중복 과금 방지와 RPO 0을 한곳에서 보장 | 단일 소유권·동기 복제·과거 소유자 차단을 모두 직접 보장해야 함 | 겹치지 않는 권한, 중앙 발급·과금과 DSP 간 멱등성을 함께 보장해야 함 | 동시 중복 소비를 사후에만 발견하므로 보장 불가 |
| **레이턴시** | 매 입찰이 중앙 원격 왕복과 복구 가능한 쓰기를 기다림 | 매 입찰이 소유자 호출과 동기 복제를 기다림 | 대부분의 예약이 DSP 로컬에서 끝남 | 로컬 예약이라 가장 짧지만 정합성을 포기함 |
| **동시성** | 캠페인별 분할은 가능하지만 한 캠페인은 같은 중앙 상태에서 직렬화 | 캠페인들은 분산할 수 있지만 한 캠페인은 소유자 하나에서 직렬화 | 같은 캠페인의 권한을 여러 DSP가 병렬 소비하나 보충 지점은 경합 가능 | 병렬 처리는 가능하지만 충돌을 예방하지 못함 |
| **장애 시 보호** | DSP 장애 복구는 단순하지만 중앙 불능이 모든 자사 입찰에 전파되어 중단 | 장애를 캠페인 단위로 격리할 수 있지만 소유권 전환 중 중단과 이중 소유 위험 | 장애 영향을 할당 권한으로 제한하지만 만료·회수와 늦은 과금 손실이 필요 | 처리는 계속할 수 있지만 로컬 유실·충돌과 초과 지출 가능 |

### A 강화 전략과 잔여 위험

| 평가축 | 강화 전략 | 강화 뒤에도 남는 위험 |
|---|---|---|
| 과금 정합성 | 경매·슬롯 단위 고유 키, 캠페인별 원자적 조건부 갱신, 잠재·확정 지출의 중앙 보존, 최초 종결 사건만 반영 | 중앙 권위 저장소의 원자성과 RPO 0에 의존 |
| 레이턴시 | 여러 슬롯을 원격 요청 한 번으로 묶고 조회 후 갱신을 하나의 조건부 처리로 축소, 연결 재사용, 경매 경로 재시도 제한 | 매 경매의 복구 가능한 원격 쓰기는 제거할 수 없음 |
| 동시성 | 캠페인별 상태 분할, 전역 잠금 제거, 짧은 트랜잭션, 캠페인별·연결 풀별 대기 상한 | 한 인기 캠페인은 같은 예산 상태에서 직렬화 |
| 장애 시 보호 | 어느 DSP에서도 같은 예약 키로 복구, 저장소 불확실 시 빠른 입찰 중단, SSP의 외부 DSP 경로 격리, 멱등 만료 작업 | 중앙 저장소 장애가 자사 DSP 전체에 전파되고 안전 확인까지 재개가 느림 |

A에서 경매별 중앙 예약을 없애기 위해 예산을 미리 나누면 C가 된다. 따라서 원격 쓰기와 인기 캠페인의 직렬화는 A의 구조적 한계다.

### C 강화 전략과 잔여 위험

C를 실제 후보로 만들려면 단순한 로컬 캐시가 아니라 중앙에서 서로 겹치지 않게 발급한 소비 권한이어야 한다.

```text
캠페인 확정 지출 + 활성 권한의 미정산 한도 <= 캠페인 총예산
```

과금이 확정되면 같은 금액을 활성 권한의 미정산 한도에서 확정 지출로 옮긴다. DSP의 모든 로컬 예약 합계는 자신이 받은 미정산 한도를 넘을 수 없다.

| 평가축 | 강화 전략 | 강화 뒤에도 남는 위험 |
|---|---|---|
| 과금 정합성 | 중앙의 원자적 권한 발급, DSP별 소유자·세대·한도, 로컬 원자적 차감, 예약 증표, 중앙 과금의 멱등 처리 | 발급·로컬 소비·과금 세 경계와 다른 DSP로 전달된 중복 요청을 함께 증명해야 함 |
| 레이턴시 | 권한이 남아 있으면 로컬 예약, 하한선에서 비동기 보충, 보충을 경매 경로에서 기다리지 않고 부족하면 입찰 포기 | 새 캠페인·권한 소진·보충 지연 시 입찰 기회 손실 |
| 동시성 | 같은 캠페인의 겹치지 않는 권한을 여러 DSP가 병렬 소비하고 권한 크기와 보충 시점을 부하에 맞게 조절 | 보충이 잦으면 중앙 발급이 다시 병목이고, 크게 나누면 예산 파편화와 페이싱 오차 증가 |
| 장애 시 보호 | 중앙 시각의 유한한 리스, 만료된 세대의 신규 예약 차단, 미갱신 권한의 자동 회수, 늦은 유효 과금의 업체 손실 처리 | 시계·갱신·만료 경합과 업체 손실이 남고, 권한 발급 또는 안전한 과금 접수가 불가능하면 입찰 중단 |

같은 경매가 다른 DSP로 재전달될 때 중앙 확인 없이 새 예약을 만들면 두 권한을 소비할 수 있다. 게이트웨이는 같은 경매·슬롯을 같은 DSP로 보내고, 기존 처리 여부가 불확실한 상태에서 다른 DSP로 넘기지 않는다. 이 때문에 DSP 장애 직후 일부 요청은 재처리하지 않고 실패시킨다.

예약 증표는 캠페인, 경매·슬롯, 금액, 권한·세대와 발급·과금 기한을 연결한다. SSP가 낙찰 근거와 함께 증표를 보존하면 DSP 로컬 상태가 사라져도 중앙은 이후 `burl`의 유효성을 검증할 수 있다.

C에서 분산 권한의 정합성·회수 문제를 없애기 위해 매 예약을 중앙에서 확인하면 A가 된다. 따라서 권한 파편화, 만료·늦은 과금과 중복 요청의 보수적 실패는 C의 구조적 한계다.

### 판정

1. **D 탈락:** 지연과 가용성은 가장 유리하지만 최우선 조건인 초과 지출 0건을 위반한다.
2. **B 미선택:** A보다 장애를 캠페인 단위로 격리할 수 있지만 원격 조정을 없애지 못한다. 그에 비해 소유권 라우팅, 동기 복제, 전환과 과거 소유자 차단을 직접 구현하는 비용이 크다.
3. **A와 C 경합:** A는 과금 정합성을 단순하게 보장하는 대신 원격 쓰기로 레이턴시 위험을 만든다. C는 레이턴시와 동시성에 유리하지만 분산 권한·만료·늦은 과금으로 정합성 구현 위험을 높인다.
4. **A 잠정 선택:** 현재 500~1,000 RPS에서 A가 p99 50ms를 만족하지 못한다는 근거가 없다. 따라서 과금 정합성을 가장 단순하게 보장하는 A를 먼저 선택한다. A가 정상·보호 요청의 50ms를 지키지 못하면 동시성이나 복잡도와 관계없이 결정을 재검토한다.

C는 폐기한 대안이 아니라 A가 성능 기준을 만족하지 못할 때 적용할 다음 후보로 남긴다.

## 결과

얻는 것:

- 예산 상한과 잠재·확정 지출의 단일 원본
- DSP 인스턴스 장애와 무관한 잠재 지출 복구
- 권한 배분·리스·회수 없이 설명 가능한 금액 상태 전이
- 높은 예산 활용률과 단순한 페이싱 입력

감수하는 것:

- 모든 자사 DSP 입찰이 원격의 복구 가능한 쓰기를 기다린다.
- 중앙 권위 저장소의 지연과 장애가 모든 자사 DSP 입찰에 영향을 줄 수 있다.
- 한 캠페인에 트래픽이 집중되면 같은 상태의 직렬화가 처리량 상한이 된다.
- 정합성을 확신할 수 없을 때 가용성보다 입찰 중단을 선택한다.
- 중앙 저장소의 원자성과 금액 상태를 다시 확신할 때까지 자사 DSP 재개가 느려질 수 있다.

## 검증과 재검토 조건

- 동시 예약과 중복 요청에서 초과 지출·중복 과금 0건
- 정상 500 RPS에서 호출자 관측 p99 50ms 이하
- 순간 1,000 RPS에서 최소 400 RPS의 p99 50ms 보호
- 단일 캠페인에 요청 100% 집중 시 같은 기준 충족
- 예약 성공 직후 DSP·저장소 인스턴스·AZ 장애에도 인정한 금액 사실의 RPO 0
- 중앙 저장소 지연·불능·응답 유실 시 불확실한 입찰 대신 명시적 중단
- `lurl`·`burl`·95초 만료 경합에서 하나의 금액 결과만 발생

합리적인 연결 관리, 자료 구조와 캠페인별 분할을 적용하고도 지연 또는 보호 처리량을 만족하지 못하면 예산 권한 선할당을 다시 비교한다.
