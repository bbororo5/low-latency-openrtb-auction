# ADR-001 분산 캠페인 예산 예약

상태: 승인됨

관련 문서: [대안 분석](../design-options/budget-reservation.md), [현재 설계](../views/budget-reservation.md)

## 맥락

여러 AZ의 자사 DSP 인스턴스가 같은 캠페인 예산으로 동시에 입찰한다. 캠페인 초과 지출은 허용하지 않으며 캠페인 선택과 예산 예약을 포함한 정상 경매는 500 RPS에서 p99 50ms 이하여야 한다. 특정 캠페인에 요청이 집중되어도 단일 승인자가 전체 처리량을 제한해서는 안 된다.

금액 정합성과 정상 지연을 입찰 기회와 자사 DSP의 금전 손실보다 우선한다.

## 결정

캠페인 예산을 중앙 권한 원장에서 서로 겹치지 않는 권한으로 나누고 DSP 인스턴스에 선할당한다. 인스턴스는 받은 권한 안에서 슬롯별 잠재 지출을 로컬로 예약하며 경매마다 중앙 승인을 받지 않는다. 잔여 권한이 부족해지기 전에 중앙에서 추가 권한을 비동기로 요청하고, 안전하게 보충하지 못하면 해당 캠페인에 입찰하지 않는다.

중앙 권한 원장은 다음 불변식을 원자적으로 지킨다.

```text
캠페인 확정 지출 + 활성 권한의 남은 금액 <= 캠페인 총예산
```

각 권한에는 캠페인, 소유자, 권한 식별자, 세대와 신규 예약 발급 기한을 둔다. 예약은 권한과 세대를 식별할 수 있고 변조를 검증할 수 있는 증표를 입찰 응답에 포함한다. 폐기된 세대는 유효한 새 예약을 만들 수 없으며 과거 소유자가 다시 살아나도 같은 권한을 재사용할 수 없다.

권한 소유 인스턴스나 AZ의 장애를 선언하면 해당 세대를 폐기한다. 폐기 전에 중앙에서 복구 가능하게 확정된 과금만 캠페인 지출로 유지하고 나머지 권한은 캠페인 가용 예산으로 반환한다. 폐기 뒤 원래 과금 기한 안에 도착한 유효한 과금 후보는 캠페인에 청구하지 않고 자사 DSP의 손실로 기록한다. 기한을 넘긴 통지는 기록만 남긴다. 폐기와 과금 접수의 선후는 중앙 원장에서 하나의 순서로 확정한다.

## 선택하지 않은 대안

- 중앙 원자적 예약은 안전하고 단순하지만 모든 경매가 원격 저장소와 AZ 간 복제 지연에 의존한다. C가 성립하지 않을 때의 기준선으로 남긴다.
- 캠페인별 단일 승인자는 예약마다 승인자와 복제본을 거치고 인기 캠페인의 단일 병목을 남기면서 리더 선출과 복제를 직접 구현해야 한다.
- 독립 예약 후 비동기 조정은 초과 지출을 사후에 발견할 뿐 예방하지 못하므로 요구사항을 위반한다.

## 결과

얻는 것:

- 정상 경매의 예산 예약을 로컬에서 처리한다.
- 캠페인 하나의 예약을 여러 인스턴스로 분산한다.
- 분산 인스턴스에서도 캠페인 예산 상한을 지킨다.
- 중앙 권한 원장이 잠시 중단되어도 이미 받은 권한 안에서는 입찰할 수 있다.

감수하는 것:

- 사용하지 않은 권한이 인스턴스별로 파편화된다.
- 권한 크기와 보충 시점이 페이싱, 보충량과 장애 손실에 영향을 준다.
- 세대 폐기, fencing, 예약 증표와 멱등 과금 처리가 필요하다.
- 장애 판단이 틀리면 정상 예약을 취소하고 자사 DSP가 비용을 부담할 수 있다.
- 중앙 권한 원장을 사용할 수 없고 로컬 권한도 소진되면 정합성을 위해 입찰을 중단한다.

## 검증

- 여러 인스턴스와 여러 슬롯의 동시 예약에서 캠페인 초과 지출 0건
- 500 RPS 정상 부하에서 호출자 기준 p99 50ms 이하
- 1,000 RPS 순간 부하에서 최소 400 RPS의 p99 50ms 보호
- 한 캠페인에 예약 요청 100% 집중 시 단일 승인자 병목 없음
- 인스턴스·AZ 장애와 과거 세대 재등장에도 권한 중복 사용 0건
- 폐기와 과금 접수 경합에서 캠페인 과금 또는 업체 손실 중 하나만 발생
- 중앙 권한 원장 장애와 권한 소진 시 초과 지출 대신 명시적 입찰 중단

권한 크기, 보충 하한선과 시간값은 이 결정을 바꾸지 않는 조정값이다. 부하·장애 시험으로 정하고 현재 설계와 실행 설정에 기록한다.
