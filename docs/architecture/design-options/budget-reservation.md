# 분산 예산 예약 대안

상태: 분석 완료

결정: [ADR-001 분산 캠페인 예산 예약](../decisions/ADR-001-distributed-budget-reservation.md)

근거: [요구사항](../../requirements/product-domain-requirements.md), [검증 기준](../../requirements/workload-data-verification.md), [ASR](../asr.md), [아키텍처 동인](../architecture-drivers.md)

## 1. 문제와 우선순위

여러 AZ의 자사 DSP 인스턴스가 같은 캠페인 총예산으로 동시에 입찰한다. 예산을 초과하지 않으면서 500 RPS의 경매를 p99 50ms 안에 끝내고, 단일 캠페인에 트래픽이 집중되어도 수평 확장할 수 있어야 한다.

우선순위는 다음과 같다.

1. 캠페인 초과 지출을 허용하지 않는다.
2. 정상 경매의 p99 50ms를 지킨다.
3. 경매마다 발생하는 원격 조정을 최소화한다.
4. 장애 중에도 금액 정합성을 지키고 빠르게 처리 능력을 복구한다.
5. 이를 위해 일부 입찰 기회와 자사 DSP의 금전 손실은 감수할 수 있다.

## 2. 비교한 모델

### A. 중앙 원자적 예약

모든 예약을 중앙 권위 저장소가 원자적으로 승인한다. 구현과 복구가 가장 단순하지만 경매마다 원격 요청과 저장소의 AZ 간 동기화가 필요하다. 서로 다른 캠페인은 분할할 수 있으나 인기 캠페인의 단일 상태에 경합이 집중된다.

### B. 캠페인별 단일 승인자

캠페인마다 하나의 활성 승인 권한을 두고 다른 인스턴스가 예약 승인을 요청한다. 범용 저장소 비용을 줄일 수 있지만 예약마다 승인자 호출과 다른 AZ 복제가 필요하다. 인기 캠페인은 단일 승인자에 집중되고 리더 선출·복제·fencing을 직접 구현해야 한다.

### C. 예산 권한 선할당

중앙 권한 원장이 캠페인 예산 일부를 서로 겹치지 않는 권한으로 나눠 DSP 인스턴스에 할당한다. 각 인스턴스는 받은 권한 안에서 로컬로 예약하고 부족할 때만 보충한다. 경매 경로의 원격 예산 호출과 인기 캠페인의 단일 승인 지점을 제거하지만 권한 파편화와 장애 회수 정책이 필요하다.

### D. 독립 예약 후 비동기 조정

각 인스턴스가 전체 잔액을 독립적으로 보고 예약한 뒤 중앙 상태를 맞춘다. 가장 빠르고 가용성이 높지만 같은 예산을 중복 사용할 수 있어 초과 지출 0건을 보장하지 못한다.

## 3. 트레이드오프

| 기준 | A. 중앙 예약 | B. 단일 승인자 | C. 권한 선할당 | D. 비동기 조정 |
|---|---|---|---|---|
| 초과 지출 방지 | 보장 | 보장 | 권한 합계로 보장 | 보장 불가 |
| 경매별 원격 조정 | 필요 | 필요 | 대부분 불필요 | 불필요 |
| 동일 캠페인 확장 | 단일 상태 경합 | 단일 승인자 경합 | 여러 인스턴스로 분산 | 분산 가능 |
| 정상 지연 | 저장소와 복제에 좌우 | 승인자와 복제에 좌우 | 로컬 예약 | 로컬 예약 |
| 장애 복구 | 비교적 단순 | 리더 전환과 fencing | 권한 폐기·반환 필요 | 초과 지출을 복구할 수 없음 |
| 예산 활용·페이싱 | 전역 상태를 보기 쉬움 | 캠페인 상태를 보기 쉬움 | 권한 파편화와 가시성 지연 | 오래된 상태와 사후 오차 |
| 구현 복잡도 | 가장 낮음 | 가장 높음 | 높음 | 낮음 |

## 4. 소거와 선택

- D는 초과 지출 금지 요구를 위반하므로 소거한다.
- B는 A보다 직접 구현 책임이 크지만 경매별 원격 조정과 인기 캠페인 병목을 제거하지 못해 선택하지 않는다.
- A는 안전하고 단순한 대체안이지만 경매마다 중앙 상태에 의존하므로 지연 최소화 우선순위에서 C보다 불리하다.
- C는 예산 상한과 로컬 예약을 함께 만족하고 동일 캠페인도 수평 확장할 수 있어 선택한다.

A가 50ms를 반드시 위반한다는 측정 근거는 아직 없다. 따라서 A는 불가능한 모델이 아니라 C의 복잡도나 예산 활용 손실이 감당할 수 없을 때 돌아갈 기준선이다.

## 5. C가 감수하는 손실

- 사용하지 않은 권한이 여러 인스턴스에 묶일 수 있다.
- 권한이 크면 페이싱 오차와 장애 시 업체 손실이 커지고, 작으면 보충 요청이 잦아진다.
- 중앙에서 보는 가용 예산은 로컬의 실제 미사용액보다 보수적이다.
- 권한 세대, 폐기, fencing과 예약 증표가 필요하다.
- 장애 시 미확정 예약을 취소하면 이미 노출된 광고 비용을 자사 DSP가 부담할 수 있다.

이 손실은 금액 정합성과 경매 지연을 우선하고 입찰 기회와 업체 손실을 후순위로 둔 결과다.

## 6. 검증할 실패점

1. 여러 인스턴스가 같은 캠페인의 권한을 동시에 보충한다.
2. 트래픽이 한 캠페인에 100% 집중된다.
3. 로컬 예약 직후 인스턴스 또는 AZ가 중단된다.
4. 장애로 잘못 판정한 과거 소유자가 다시 요청을 처리한다.
5. 중앙 권한 원장이 중단된 상태에서 로컬 권한이 소진된다.
6. 권한 폐기와 `burl` 접수가 동시에 일어난다.
7. 중복·지연·위조된 예약 증표가 전달된다.
