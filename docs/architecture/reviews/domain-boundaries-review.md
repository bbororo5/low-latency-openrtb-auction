# 도메인 경계 관점 검토

검토 대상: [아키텍처 동인](../architecture-drivers.md)

## 결론

경매, 캠페인 입찰, 예산은 의미와 불변식이 달라 논리적 책임 경계를 나누는 것이 유효하다. 반면 과금 연결은 독립된 도메인이라기보다 SSP의 경매 근거와 자사 DSP의 예산 상태를 잇는 업무 흐름이며, 검증 자료는 도메인 경계가 아니라 공통 운영 관심사다.

## 경계 판정

| 경계 | 소유하는 것 | 소유하지 않는 것 |
|---|---|---|
| SSP 경매·정산 | 요청 검증, DSP별 응답 판정, 마감, 낙찰, 렌더링 검증, 과금·미과금 확정 | 자사 DSP의 캠페인과 광고주 예산 |
| 외부 DSP 연동 | 외부 응답을 내부 경매 의미로 변환하는 규칙 | 외부 회사의 캠페인·예산 상태 |
| 자사 DSP 입찰 | 실행 데이터, 후보 검색, 캠페인·소재 선택, 입찰가 | 경매 최종 낙찰과 공급자 노출 판정 |
| 자사 DSP 예산 | 잠재 지출, 확정 지출, 한도와 금액 상태 전이 | SSP 경매 정책과 외부 DSP 상태 |

SSP와 자사 DSP는 서로 다른 회사이므로 독립 배포·독립 저장소를 사용한다. 자사 DSP 내부의 입찰과 예산 경계를 같은 프로세스·저장소에 둘지는 이후 결정한다.

## 경계를 넘는 사실

| 흐름 | 전달할 사실 | 받는 쪽의 책임 |
|---|---|---|
| SSP → 외부·자사 DSP | 경매·광고 영역·마감·최저가·통화 | 입찰 여부와 가격 결정 |
| 자사 DSP 입찰 → 예산 | 캠페인·광고주·노출 1건 금액·입찰 식별자 | 한도 안에서 잠재 지출 확보 |
| DSP → SSP | 요청과 광고 영역에 연결된 입찰 | 유효성 판정과 낙찰 결정 |
| 공급자 역할 → SSP | 렌더링 주장과 사건 식별자 | 원래 낙찰과 연결하고 과금·미과금 확정 |
| SSP → 자사 DSP 예산 | 낙찰·패찰과 최종 과금·미과금 사실 | 잠재 지출 유지·해제 또는 확정 지출 전환 |

경계를 넘는 메시지는 상대 경계의 내부 자료구조가 아니라 이미 판정된 사실과 식별자만 포함해야 한다.

## 용어와 불변식

- `입찰`은 DSP의 구매 제안이고 `낙찰`은 SSP의 경매 결과다.
- `렌더링 통지`는 공급자 역할이 주장한 사실이고 `과금·미과금`은 SSP가 검증해 확정한 사실이다.
- `최종 정산`은 SSP가 낙찰과 연결해 자사 DSP에 전달한 상태 전이 입력이다.
- `잠재 지출`과 `확정 지출`의 불변식은 자사 DSP 예산 경계만 변경할 수 있다.
- 캠페인 실행 데이터는 자사 DSP 입찰 경계가 사용하는 파생 모델이며 별도 도메인으로 볼 근거는 아직 없다.

## 동인 수정 사항

- 현재 책임 경계 표의 `과금 연결`은 SSP의 낙찰 연결 책임과 자사 DSP 예산의 금액 전이 책임으로 분리해 표현해야 한다.
- `검증 자료`는 책임 경계 후보에서 제외하고 모든 경계를 가로지르는 검증 가능성으로 다뤄야 한다.
- 자사 DSP 입찰과 예산은 논리적으로 구분하되 입찰 전에 잠재 지출을 확보해야 한다는 하나의 업무 규칙으로 연결해야 한다.
- SSP와 자사 DSP가 상대의 내부 금액·캠페인 모델을 직접 읽거나 변경하는 구조는 피해야 한다.

## ADR로 넘길 질문

- 자사 DSP 입찰과 예산 책임을 같은 배포 단위에 둘지 분리할지 판단 기준은 무엇인가?
- 입찰 결정과 잠재 지출 확보의 성공·실패를 어떤 계약으로 묶을 것인가?
- SSP는 렌더링과 최종 정산을 판정하기 위해 낙찰 정보 중 무엇을 소유해야 하는가?
- SSP에서 자사 DSP로 전달하는 금액 사건의 최소 계약은 무엇인가?
- 캠페인 원본 변경을 실행 데이터로 변환하는 책임은 어디에 둘 것인가?
