# 진화적 아키텍처 관점 검토

검토 대상: [아키텍처 동인](../architecture-drivers.md)

## 결론

통신 형식 독립성과 재현 가능한 검증은 유효한 동인이다. 다만 변화 가능성을 통신 표현에만 한정하면 부족하다. 데이터 계약, 금액 표현, 저장 상태가 순차 배포 중에도 이전 버전과 공존해야 한다는 압력이 추가로 드러난다.

## 유지해야 할 성질

다음은 구현과 배포 구조가 바뀌어도 유지되어야 한다.

- 제한 시간을 넘긴 입찰은 낙찰되지 않는다.
- 같은 입력과 유효 후보 집합은 같은 낙찰과 캠페인 선택을 만든다.
- 예산 초과, 중복 과금, 인정된 금액 효과 유실은 0건이다.
- 인스턴스와 AZ 장애에서 전체 서비스 중단이 발생하지 않는다.
- 성능·장애·금액 시험을 같은 조건으로 반복할 수 있다.

이 성질은 문서상 원칙이 아니라 자동 시험과 운영 지표로 계속 판정되어야 한다.

## 확인된 변화 방향

| 변화 | 지켜야 할 경계 |
|---|---|
| HTTP/JSON → 이진 통신 | 경매·금액 의미와 도메인 시험을 변경하지 않는다. |
| 외부 DSP 추가·교체 | 한 DSP의 계약과 장애가 다른 DSP 및 경매 핵심으로 번지지 않는다. |
| 캠페인 공급·조회 방식 변경 | 캠페인 선택 규칙과 데이터 버전별 결정성을 유지한다. |
| 금액 표현·저장 방식 변경 | 기존 잠재·확정 지출과 사건 식별 의미를 보존한다. |
| 복제본·AZ 구성 변경 | 인스턴스 수가 도메인 규칙과 금액 효과를 바꾸지 않는다. |
| 순차 배포 | 이전 버전과 새 버전이 공존하는 동안 통신·상태 계약이 호환된다. |

## 지속적으로 판정할 기준

| 성질 | 판정 수단 |
|---|---|
| 경매 정확성 | 고정 입력에 대한 낙찰·동률·마감 시험 |
| 금액 정합성 | 동시·중복·역순·만료 시험과 전후 예산 대조 |
| 성능 | 500 RPS p99 50ms와 1,000 RPS 순간 부하 시험 |
| 회복 | 인스턴스·AZ 장애 및 과부하 후 회복 시간 |
| 경계 보존 | SSP, 입찰, 예산 코드의 의존 방향 검사 |
| 계약 호환성 | 이전·새 통신 및 저장 상태를 섞은 순차 배포 시험 |

## 동인 수정 사항

- `동인-06`은 통신 표현뿐 아니라 순차 배포 중 계약 호환성과 자동 판정을 포함해야 한다.
- 변화 가능성이 있다는 이유만으로 추상화하지 않고 위 표에 확인된 변화만 격리 대상으로 삼아야 한다.
- 금액 단위, 사건 식별자, 데이터 소유권처럼 바꾸기 어려운 결정은 초기 ADR에서 우선해야 한다.
- 통신 직렬화 도구처럼 교체 가능한 결정은 도메인 경계 밖에 두고 필요할 때 변경해야 한다.

## ADR로 넘길 질문

- 이전 버전과 새 버전이 공존할 때 어떤 통신·사건·저장 계약을 호환해야 하는가?
- 변경하기 어려운 데이터 소유권, 사건 식별자, 금액 단위를 어떤 순서로 결정할 것인가?
- 통신 표현과 저장 기술이 도메인 모델로 새어 들어오지 않게 어떤 경계를 둘 것인가?
- 성능·금액·회복 기준 중 어떤 것을 배포 차단 조건으로 자동화할 것인가?
- 캠페인 데이터 버전 변경 중 같은 경매를 어느 버전으로 판정할 것인가?
