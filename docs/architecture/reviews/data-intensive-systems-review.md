# 데이터 중심 시스템 관점 검토

검토 대상: [아키텍처 동인](../architecture-drivers.md)

## 결론

금액 정합성과 장애 후 상태 보존은 핵심 동인이 맞다. 다만 현재 문서는 데이터마다 원본성, 수명, 정합성, 복구 기준이 다르다는 사실만 적었고 **언제 상태를 인정하며 어디까지 보존해야 하는지**는 충분히 드러내지 못했다.

## 데이터별 판정

| 데이터 | 판정 | 이유 |
|---|---|---|
| 캠페인 실행 데이터 | 다시 만들 수 있는 파생 데이터 | 원본 시스템이 따로 있고 조회 경로에 최적화된 사본이므로 갱신 지연과 재구축 기준이 필요하다. |
| 경매 진행 상태 | 짧게 유지되는 작업 상태 | 마감 뒤에는 결과를 바꿀 수 없지만 진행 중 유실은 해당 경매 실패가 될 수 있다. |
| 공개된 경매 결과와 연결 정보 | 정산 종료까지 필요한 근거 | 렌더링 통지와 최종 결과를 원래 경매·입찰에 연결하고 DSP 접수까지 복구 가능해야 한다. |
| 잠재·확정 지출 | 자사 DSP의 금액 원본 | 동시 갱신과 장애에서도 광고주·캠페인 한도를 함께 지켜야 한다. |
| 금액 사건 | 상태 전이의 근거 | 중복·역순·충돌을 판정하고 장애 후 재처리해도 같은 결과를 내야 한다. |
| 지표와 로그 | 검증용 파생 데이터 | 금액 원본을 대신할 수 없으며 유실 여부가 예산 상태를 바꾸면 안 된다. |

## 확인된 압력

- 한 입찰의 잠재 지출 확보와 입찰 응답 사이에는 성공으로 인정하는 경계가 필요하다.
- SSP의 렌더링 접수와 DSP의 최종 결과 접수에는 각각 성공으로 인정하는 경계가 필요하다.
- 광고주 한도와 캠페인 한도는 하나의 입찰에서 함께 보존되어야 한다.
- 상위 1% 광고주에 갱신이 집중되므로 평균 처리량만으로 동시성 방식을 판단할 수 없다.
- 과금이 늦게 도착해도 일일 예산은 경매 수신 시각의 날짜에 귀속되어야 한다.
- 렌더링 접수와 60초 미과금 판정이 경합해도 하나의 최종 상태만 허용해야 한다.
- 복제 지연이나 통신 단절 중에도 오래된 예산으로 초과 입찰해서는 안 된다.

## 동인 수정 사항

- `동인-03`의 “인정된 금액 효과”가 언제부터 인정되는지 ADR 전에 명시해야 한다.
- `동인-05`는 단순히 데이터를 성격에 맞게 다루는 수준이 아니라 원본·파생 구분, 보존 기간, 정합성 경계, 복구 기준을 결정하도록 요구해야 한다.
- 경매 결과 전체를 영구 보존할 필요는 없지만 정산에 필요한 최소 정보는 DSP가 최종 결과를 복구 가능하게 접수할 때까지 보존해야 한다.
- 정확히 한 번 전달을 전제하지 말고, 중복 전달에서도 금액 효과가 한 번만 발생하도록 해야 한다.

## ADR로 넘길 질문

- 잠재 지출 확보와 금액 사건 처리는 어느 시점에 성공으로 응답할 것인가?
- 광고주·캠페인 한도를 한 번의 입찰에서 어떤 정합성 경계로 보호할 것인가?
- 낙찰 결과와 과금 연결 정보는 무엇을 얼마 동안 보존할 것인가?
- 캠페인 실행 데이터의 갱신 지연과 재구축 중 입찰 가능 여부를 어떻게 정할 것인가?
- 중복·역순·60초 경계 경합을 어떤 사건 식별자와 상태 전이로 판정할 것인가?
- AZ 단절과 복구 뒤 금액 원본을 어떻게 대조할 것인가?
