## 2. 아키텍처 원칙

- SSP와 프로젝트 DSP는 서로 다른 회사이며 저장소와 트랜잭션을 공유하지 않는다.
- 두 회사는 각각 두 리전에 능동 배치하고 각 리전의 실행 컨테이너를 여러 AZ에 복제한다.
- SSP는 세 DSP를 병렬 호출하며 회사별 시간·동시성·연결 자원을 격리한다.
- SSP 경매와 렌더링·통지는 같은 애플리케이션의 실행 자원을 격리한다. 독립 배포 여부는 아직 결정하지 않았다.
- 프로젝트 DSP의 입찰은 인스턴스 로컬 캠페인 자료와 로컬 예산 권한만 사용한다.
- 프로젝트 DSP의 입찰과 예산 처리는 같은 애플리케이션 프로세스 안에서 실행 자원을 격리한다.
- 캠페인 예산은 전역 책임액, 리전 책임액과 DSP 로컬 권한으로 겹치지 않게 나눈다.
- 전역 책임 원장과 각 리전 예산 원장은 물리 저장소 배포를 분리해 장애와 합의 범위를 공유하지 않는다.
- `lurl`·`burl` 금액 사건은 SSP 발신함과 DSP 수신함을 거쳐 중복 전달되어도 한 번만 반영한다.
- `nurl`은 금액을 바꾸지 않는 경매 결과 사건으로 금액 원본과 구분한다.
- 외부에 성공을 알린 사실은 리전 하나를 잃어도 복구할 수 있어야 한다.

## 3. 데이터 권위

| 데이터 | 권위 | 보존 기준 |
|---|---|---|
| 경매·낙찰·렌더링·통지 | SSP | 성공 사실과 발신함은 RPO 0 |
| 캠페인 실행 자료 | 프로젝트 DSP | 버전이 있는 로컬 조회 사본 배포 |
| 총예산·확정 지출 집계·전역 예비액·리전 책임액 | 프로젝트 DSP 전역 책임 원장 | 책임 이전과 인정된 금액 사실은 강하게 보존 |
| DSP 권한·잠재 지출 | DSP 입찰 인스턴스와 리전 예산 원장 | 로컬 예약, 장애 시 상위 권한 격리 |
| 접수한 `lurl`·`burl`과 처리 상태 | 프로젝트 DSP 금액 사건 저장소 | 접수 성공 사실과 복구 가능한 금액 근거는 RPO 0 |
| 접수한 `nurl` | 프로젝트 DSP 경매 결과 자료 | 금액을 바꾸지 않으며 추적·분석 필요에 맞춰 보존 |
| 지표·로그·페이싱 투영 | 각 회사 | 파생 자료이며 금액 원본이 아님 |

## 4. ADR 대조 결과

| ADR | Level 2와 Deployment에 반영한 구조 |
|---|---|
| ADR-001 분산 예산 예약 | DSP 애플리케이션의 로컬 권한과 입찰 경로 밖의 원장 관계 |
| ADR-002 다중 리전 원장 | 전역 책임 원장, 리전별 독립 원장과 비동기 복구 사본 |
| ADR-003 리전 예산 이전 | DSP 애플리케이션의 배경 처리가 전역·리전 원장에 접근하고 입찰은 이전을 기다리지 않음 |
| ADR-004 경매 실행 경로 | SSP 애플리케이션, DSP 게이트웨이·DSP 애플리케이션과 경로별 수용량 제한 |
| ADR-005 금액 사건 복구 | SSP 애플리케이션의 렌더링·통지 실행 자원과 성공 사실 저장소, DSP 애플리케이션과 금액 사건 저장소 |
| ADR-006 다중 리전 배치 | 두 회사의 독립 전역 진입, 두 능동 리전, 리전별 다중 AZ 인스턴스 |

현재 컨테이너안은 다음 기준으로 구성했다.

- 저지연 경매와 재시도 가능한 통지는 실행 자원을 격리하되 독립 배포는 근거가 생길 때만 선택한다.
- DSP 입찰과 예산 제어는 책임과 실행 자원만 구분하고 독립 배포 단위로 만들지 않는다.
- 전역 책임 원장과 각 리전 예산 원장은 물리 저장소 배포를 분리한다. 금액 사건과 캠페인·페이싱의 세부 경계는 [데이터 관점](views/data.md)에 기록한다.
- 내부 계산 단계와 저장소 스키마는 독립 배포 근거가 없으므로 컨테이너로 만들지 않는다.

## 5. 기술 선택 경계

현재 문서는 제품과 구현 방식을 정하지 않는다. 다음 단계에서 기술 후보가 아래 능력을 만족하는지 비교한다.

- 다중 리전 RPO 0과 SSP 전체 p99 50ms의 동시 충족
- 책임액의 조건부 원자 이전과 단일 소유권 차단
- 리전 내부 고가용과 높은 동시 권한 발급
- 낮은 할당·GC 부담의 로컬 예약과 캠페인 조회
- 내구성 있는 발신함·수신함, 멱등 재처리와 적체 제한
- 절대 기한 전파, 연결 재사용과 회사별 자원 격리

기술 후보는 데이터의 원본성과 복구 계약을 만족해야 한다. 물리 저장소 경계를 바꾸는 선택은 데이터 관점과 관련 ADR을 함께 다시 검토한다.
