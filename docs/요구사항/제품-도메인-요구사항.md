# 제품·도메인 요구사항

상태: 기준선 2.0

수치와 합격 기준: [부하·데이터·검증 기준](부하-데이터-검증-기준.md)

이 프로젝트는 제한 시간 경매, 실제 캠페인 선택, 예산 정합성, 장애 지속성을 다룬다. 광고 플랫폼 전체는 만들지 않는다.

## 1. 목표

| ID | 목표 | 성공 기준 |
|---|---|---|
| `목표-01` | 제한 시간 안에 복수 DSP 응답으로 올바른 낙찰자를 결정한다. | 늦거나 잘못된 입찰이 낙찰되지 않고 하나의 DSP 실패가 전체 실패로 퍼지지 않는다. |
| `목표-02` | 자사 DSP가 캠페인·소재·예산을 보고 입찰한다. | 적격 후보만 선택하고 후보가 없으면 `NO_BID`한다. |
| `목표-03` | 동시 입찰과 중복·역순 금액 이벤트에서 예산을 보존한다. | 예산 초과, 음수 잔액, 중복 과금이 0건이다. |
| `목표-04` | 애플리케이션 인스턴스 또는 하나의 AZ가 중단되어도 서비스를 지속한다. | SSP와 자사 DSP의 전체 중단이 없고 인정된 금액 효과가 보존된다. |
| `목표-05` | 성능·장애·정합성을 재현하고 설명한다. | 동일한 부하·데이터·장애에서 응답 시간, 처리량, 정확성, 복구를 함께 검증한다. |

## 2. 범위

| 역할 | 책임 |
|---|---|
| 공급자 | 광고 기회와 제한 시간을 SSP에 전달하고 결과를 받는다. |
| SSP | 요청 검증, DSP 호출, 입찰 검증, 낙찰자 결정, 결과 연결을 담당한다. |
| 자사 DSP | 캠페인과 예산을 기준으로 입찰 여부와 가격을 결정한다. |
| 외부 DSP | 서로 다른 회사를 나타내며 독립적으로 응답한다. 내부 구조는 범위 밖이다. |
| 광고주 | 자사 DSP에 캠페인과 예산을 맡긴다. |
| 과금 이벤트 공급자 | 낙찰된 광고가 과금 가능한 노출이 되었음을 알린다. |

포함 범위:

- 하나의 광고 영역에 대한 OpenRTB 2.6 하위 규격 경매
- 자사 DSP와 두 외부 DSP로의 논리적 확산 호출
- 유효 입찰, 무입찰, 시간 초과, 잘못된 입찰, 오류 분류
- 1차 가격 경매와 결정적 동률 처리
- 자사 DSP의 캠페인 검색·선택·입찰 가격 결정
- 잠재 지출, 낙찰·패찰·과금·만료와 예산 불변식
- 인스턴스·AZ 장애, 과부하, 순차 배포의 검증

제외 범위:

- 광고 표시 SDK, 클릭·기여 추적, 사용자 식별·프로파일링
- 캠페인·소재 관리 화면과 원본 관리 기능
- 머신러닝 입찰 최적화, 고급 속도 조절, 빈도 제한
- 광고 사기, 가시성, 소재 심사, 브랜드 안전성
- 복수 통화, 세금, 청구·환불, 대외 정산·회계
- 외부 사업자 온보딩·공개망 보안 운영과 외부 DSP 내부
- 다중 리전 재해 복구, 쿠버네티스, 자동 확장 제어면

## 3. 핵심 규칙

### 3.1 경매

- `입찰마감 = SSP 요청 수신 시각 + 유효 tmax`이다.
- DSP에는 입찰 마감까지 남은 시간만 전달한다.
- 마감 이후 응답은 결과를 바꾸지 않는다.
- 각 DSP 호출은 `VALID_BID`, `NO_BID`, `TIMEOUT`, `INVALID_BID`, `ERROR` 중 하나로 끝난다.
- 유효 입찰 중 CPM이 가장 높은 입찰이 낙찰한다. 동률은 `dspId`, `bidId` 순으로 결정한다.

### 3.2 캠페인 선택

1. 운영 기간, 매체·광고 형식, 크기, 게재 위치·분류, 최저가, 통화로 후보를 거른다.
2. 광고주와 캠페인의 사용 가능 예산을 확인한다.
3. `금일 과금 노출 수 / 금일 목표 노출 수`가 가장 낮은 캠페인을 선택한다.
4. 동률은 `campaignId`, `creativeId` 순으로 결정한다.
5. 적격 후보가 없으면 `NO_BID`한다.

### 3.3 금액과 예산

- 통화는 KRW 하나다.
- 입찰가와 최저가는 CPM이며 과금 노출 1건의 금액은 CPM의 `1/1000`이다.
- CPM은 소수점 셋째 자리까지 허용하며 더 정밀한 값은 거절한다.
- 누적·비교는 이진 부동소수점 오차로 결과가 달라지지 않아야 한다. 내부 자료형은 결정하지 않는다.
- 일일 예산의 날짜는 경매 수신 시각의 `Asia/Seoul` 날짜다.

```text
광고주 확정 지출 + 잠재 지출 <= 충전액
캠페인 누적 확정 지출 + 잠재 지출 <= 누적 한도
캠페인 일일 확정 지출 + 잠재 지출 <= 일일 한도
```

| 이벤트 | 금액 효과 |
|---|---|
| 입찰 제시 | 입찰가의 노출 1건 금액을 잠재 지출로 잡는다. |
| 낙찰 | 낙찰을 확정하지만 확정 지출은 늘리지 않는다. |
| 패찰 | 잠재 지출을 해제한다. |
| 과금 노출 | 낙찰가를 확정 지출로 한 번만 반영한다. |
| 만료 | 끝나지 않은 입찰의 잠재 지출을 해제한다. |

동일한 `auctionId + bidId + eventType`은 금액에 한 번만 영향을 준다. 낙찰과 과금의 순서가 바뀌어도 최종 상태는 같아야 한다. 패찰과 과금처럼 모순된 종료 이벤트는 `MONEY_EVENT_CONFLICT`로 격리한다.

### 3.4 장애 범위

- SSP 인스턴스 하나, 자사 DSP 인스턴스 하나, AZ 하나의 중단이 전체 중단을 만들면 안 된다.
- 순차 배포는 전체 중단과 금액 불일치 없이 끝나야 한다.
- 장애 전에 인정한 금액 효과는 유실·중복되면 안 된다.
- 장애 순간의 처리 중 경매는 일부 실패할 수 있지만 예산 불변식은 깨지면 안 된다.
- 리전 전체 장애는 범위 밖이다.

## 4. 기능 요구사항

| ID | 요구사항 |
|---|---|
| `기능-01` | SSP는 요청을 검증하고 OpenRTB 요청을 만든다. 실행할 수 없는 요청은 DSP 호출 전에 거절한다. |
| `기능-02` | 선택한 각 DSP에 의미상 같은 요청을 보내고 각 호출을 하나의 종료 결과로 분류한다. |
| `기능-03` | 마감 안의 입찰만 요청, 가격, 통화, 매체, 소재 기준으로 검증하고 낙찰자 또는 무낙찰을 결정한다. |
| `기능-04` | 하나의 DSP 지연·오류를 격리하고 남은 유효 입찰로 경매를 완료한다. |
| `기능-05` | 자사 DSP는 요청 조건과 예산을 만족하는 캠페인·소재를 결정적으로 선택한다. |
| `기능-06` | 자사 DSP는 입찰을 반환하기 전에 광고주·캠페인 예산 범위의 잠재 지출을 확보한다. |
| `기능-07` | 낙찰·패찰·과금·만료를 중복과 순서 변경에 안전하게 처리한다. |
| `기능-08` | 경매, 입찰, 캠페인, 가격, 통화, 금액 이벤트의 연결 관계를 보존한다. |

## 5. 품질 요구사항

| ID | 요구사항 |
|---|---|
| `품질-01` | 정상·시간 초과·과부하 검증에서 낙찰 정확성과 응답 시간 기준을 함께 만족한다. |
| `품질-02` | DSP 시간 초과와 부분 실패를 경매 제한 시간 안에서 격리한다. |
| `품질-03` | 동시성, 중복, 역순, 장애에서 예산 불변식과 한 번의 금액 효과를 보존한다. |
| `품질-04` | 스레드, 대기열, 연결, 처리 중 작업에 상한을 두고 과부하 후 재시작 없이 회복한다. |
| `품질-05` | 같은 입력과 유효 후보 집합은 처리 순서와 무관하게 같은 결과를 만든다. |
| `품질-06` | 인스턴스 하나와 AZ 하나의 장애가 SSP 또는 자사 DSP 전체 중단을 만들지 않는다. |
| `품질-07` | 장애 전에 인정한 금액 효과의 RPO는 0이다. |
| `품질-08` | 경매, DSP, 캠페인 검색, 금액, 자원 포화, 장애 전환을 무한히 늘지 않는 분류값으로 관측한다. |
| `품질-09` | JSON에서 이진 표현으로 바꾸어도 경매·금액 의미와 합격 시험은 달라지지 않는다. |

## 6. 제약과 가정

제약:

- SSP–DSP 간 의미는 OpenRTB 2.6의 명시적 하위 규격을 따른다.
- 공급자 API는 프로젝트 전용이며 OpenRTB 객체를 직접 노출하지 않는다.
- 첫 버전은 하나의 광고 영역, 1차 가격, 배너 우선, KRW 단일 통화다.
- 실행 환경은 Java 21이다.
- AWS 단일 리전에서 AZ 하나의 장애를 견뎌야 하며 SSP와 자사 DSP의 단일 애플리케이션 인스턴스 배포는 허용하지 않는다.
- HTTP/JSON은 초기 표현일 뿐 영구 제약이 아니다.

가정:

- 외부 DSP의 내부 캠페인과 예산은 알 수 없다.
- 매체·캠페인 원본은 별도 시스템이 관리하며 재현 가능한 실행용 데이터를 제공한다.
- 과금 이벤트 공급자는 경매·입찰 식별자, 낙찰가, 통화를 신뢰할 수 있게 제공한다.
- 일일 예산의 날짜 경계는 `Asia/Seoul`이다.

구현이 어렵다는 이유로 기준을 낮추지 않는다. 예산 불변식이나 장애 범위를 바꾸려면 ADR이 아니라 이 문서부터 바꾼다.
